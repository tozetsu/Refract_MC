Refract 1.2.0 - Release Notes

Main update
- Refract is moving from the old Electron desktop app to the new Tauri desktop app.
- The Tauri app keeps the same app identity: com.refract.
- Existing launcher data continues to use the same %APPDATA%/Refract folder.
- Existing instances, configuration, Java settings, themes, worlds, screenshots, options, and server lists are preserved.
- Microsoft accounts need a one-time sign-in again because the new Tauri app stores tokens in a different secure vault.
- Offline accounts continue to work.

Auto-update migration
- Added a final Electron-to-Tauri bridge app.
- Existing Electron installs can update to the bridge through the old Electron updater feed.
- The bridge checks the Tauri latest.json manifest on GitHub Releases.
- The bridge downloads the Windows Tauri installer for the latest version.
- The bridge starts the Tauri installer silently and then quits.
- After Tauri is installed once, future updates are handled by the Tauri updater.

Tauri updater
- Tauri updater is configured to read:
  https://github.com/RefractMC/Refract_MC/releases/latest/download/latest.json
- Tauri builds create updater artifacts for signed releases.
- The in-app updater flow checks for updates, shows an update banner, downloads on user action, and relaunches to install.

Release automation
- Added a new Electron bridge package at apps/electron-bridge.
- Added a new root command:
  pnpm build:electron-bridge
- Added a GitHub Actions workflow:
  .github/workflows/release-electron-bridge.yml
- The Electron bridge workflow uploads these assets to an existing release:
  latest.yml
  Refract Setup <version>.exe
  Refract Setup <version>.exe.blockmap
- The Tauri release still provides latest.json and the Tauri installer artifacts.

Developer and documentation updates
- Updated release documentation with the preferred automatic Electron-to-Tauri migration path.
- Updated contributor documentation to mention the Electron bridge package and build command.
- Updated pnpm workspace configuration to allow Electron's install build step.
- Updated the lockfile for the new Electron bridge dependencies.

Release process for this version
1. Publish the signed Tauri release for v1.2.0.
2. Make that Tauri release public and latest.
3. Run the Release (Electron bridge) workflow with the same tag.
4. Old Electron users receive latest.yml and update to the bridge.
5. The bridge installs Tauri.
6. Future app updates are delivered by Tauri.
